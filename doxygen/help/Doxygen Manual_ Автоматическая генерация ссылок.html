Автоматическая генерация ссылок

Содержание

  • Ссылки на веб-страницы и почтовые адреса
  • Ссылки на классы
  • Ссылки на файлы
  • Ссылки на функции
  • Ссылки на других участников
  • typedefs

В большинстве систем документации есть специальные разделы «Смотрите также», в которые могут быть вставлены ссылки на другие части документации. Несмотря на то, что в doxygen также есть команда для запуска такого раздела (см. раздел \sa), он позволяет размещать такие ссылки в любом месте документации. Для документации вместо ссылки пишется ссылка на номер страницы. Кроме того, индекс в конце документа можно использовать для быстрого поиска документации по члену, классу, пространству имен или файлу. Для man-страниц справочная информация не создается.

В следующих разделах показано, как создавать ссылки на различные документированные сущности в исходном файле.

Ссылки на веб-страницы и почтовые адреса

Doxygen автоматически заменит любые URL-адреса и почтовые адреса, найденные в документации, ссылками (в HTML). Чтобы вручную указать текст ссылки, используйте HTML-тег '': a

<a href="linkURL">link text</a>

которые будут автоматически переведены в другие выходные форматы с помощью doxygen.

Ссылки на классы

Все слова в документации, которые соответствуют документированному классу и содержат хотя бы один символ не в нижнем регистре, будут автоматически заменены ссылкой на страницу, содержащую документацию класса. Если вы хотите, чтобы слово, соответствующее документированному классу, не заменялось ссылкой, вы должны поставить % перед словом. Чтобы создать ссылку на символ в нижнем регистре, используйте \ref.

Ссылки на файлы

Все слова, содержащие точку (), которая не является последним символом в слове, считаются именами файлов. Если слово действительно является именем документированного входного файла, будет автоматически создана ссылка на документацию этого файла..

Ссылки на функции

Ссылки на функции создаются, если встречается один из следующих шаблонов:

  1. <functionName>"("<argument-list>")"
  2. <functionName>"()"
  3. "::"<functionName>
  4. (<className>"::")n<functionName>"("<argument-list>")"
  5. (<className>"::")n<functionName>"("<argument-list>")"<modifiers>
  6. (<className>"::")n<functionName>"()"
  7. (<className>"::")n<functionName>

где n>0.

Примечание 1:
Аргументы функции должны быть указаны с правильными типами, т.е. 'fun(const std::string&,bool)' или '()', чтобы соответствовать любому прототипу.
Примечание 2:
Модификаторы функций-членов (такие как 'const' и 'volatile') необходимы для идентификации цели, т.е. 'func(int) const' и 'func(int)' предназначены для различных функций-членов.
Примечание 3:
Для совместимости с Javadoc вместо :: в приведенных выше шаблонах можно использовать #.
Примечание 4:
В документации к классу, содержащему член foo, ссылка на глобальную переменную делается с помощью "::foo", тогда как #foo будет ссылаться на член.

Для неперегруженных членов список аргументов может быть опущен.

Если функция перегружена и не указан соответствующий список аргументов (т.е. используется шаблон 2 или 6), будет создана ссылка на документацию одного из перегруженных членов.

Для функций-членов область видимости класса (используемая в шаблонах с 4 по 7) может быть опущена, если:

  1. Шаблон указывает на документированный элемент, принадлежащий тому же классу, что и блок документации, содержащий шаблон.
  2. Класс, соответствующий блокам документации, содержащим шаблон, имеет базовый класс, содержащий документированный член, соответствующий шаблону.

Ссылки на других участников

Все эти сущности могут быть связаны таким же образом, как описано в предыдущем разделе. Для ясности в данном случае рекомендуется использовать только паттерны 3 и 7.

Пример:
/*! \file autolink.cpp
Тестирование автоматической генерации ссылок.
Ссылка на член класса Autolink_Test: Autolink_Test::member,
Более конкретные ссылки на каждый из перегруженных элементов:
Autolink_Test::member(int) и Autolink_Test#member(int,int)
Ссылка на защищенную переменную-член Autolink_Test: Autolink_Test#var,
Ссылка на глобальный тип перечисления #GlobEnum.
Ссылка на определение #ABS(x).
Ссылка на деструктор класса Autolink_Test: Autolink_Test::~Autolink_Test,
Ссылка на typedef ::B.
Ссылка на тип перечисления Autolink_Test::EType
Ссылка на некоторые значения перечисления Autolink_Test::Val1 и ::GVal2
*/
/*!
Так как данный блок документации относится к классу, Autolink_Test нет ссылки на
Autolink_Test генерируется.
Существует два способа связывания с конструктором: #Autolink_Test и Autolink_Test().
Ссылки на деструктор: #~Autolink_Test и ~Autolink_Test().
Ссылка на член в этом классе: member().
Более конкретные ссылки на каждый из перегруженных элементов:
member(int) и member(int,int).
Ссылка на переменную #var.
Ссылка на глобальный typedef ::B.
Ссылка на глобальный тип перечисления #GlobEnum.
Ссылка на определение ABS(x).
Ссылка на переменную \link #var использование другого text\endlink в качестве ссылки.
Ссылка на тип перечисления #EType.
Ссылка на некоторые значения перечисления: \link Autolink_Test::Val1 Val1 \endlink и ::GVal1.
И последнее, но не менее важное: ссылка на файл: autolink.cpp.
\sa Внутри раздела см. также проверяется любое слово, поэтому EType,
Val1, GVal1, ~Autolink_Test и member будут заменены ссылками в HTML.
*/
класс Autolink_Test
{
Публичные:
Autolink_Test(); //!< конструктор
~Autolink_Test(); //!< деструктор
void member(int); /**< Функция-член. Подробности. */
void member(int,int); /**< Перегруженная функция-член. Подробности*/
/** Тип перечисления. Подробнее */
Перечисление EType {
Val1, /**< значение перечисления 1 */
Val2 /**< значение перечисления 2 */
};
Защищено:
int var; /**< Переменная-член */
};
/*! подробности. */
Autolink_Test::Autolink_Test() { }
/*! подробности. */
Autolink_Test::~Autolink_Test() { }
/*! Глобальная переменная. */
int globVar;
/*! Глобальное перечисление. */
Перечисление GlobEnum {
GVal1, /*!< глобальное значение перечисления 1 */
GVal2 /*!< глобальное значение перечисления 2 */
};
/*!
* Определение макроса.
*/
#define ABS(x) (((x)>0)?( x):-(x))
typedef Autolink_Test В;
/*! \fn typedef Autolink_Test B
* Определение типа.
*/

typedefs

Определения типов, включающие классы, структуры и объединения, такие как 

typedef struct StructName TypeName

создайте псевдоним для StructName, чтобы ссылки на StructName создавались при обнаружении либо самого StructName, либо TypeName.

Пример:
/*! \file restypedef.cpp
* Пример разрешения typedefs.
*/
/*! \struct CoordStruct
* Координатная пара.
*/
Структура CoordStruct (КоордСтруктура)
{
/*! Координата x */
число с плавающей запятой x;
/*! Координата y */
число с плавающей запятой y;
};
/*! Создает имя типа для CoordStruct */
typedef CoordStruct — коорд;
/*!
* Эта функция возвращает сложение \a c1 и \a c2, т.е.:
* (c1.x+c2.x,c1.y+c2.y)
*/
Добавление координат (Координата c1,Координата c2)
{
}
